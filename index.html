Question 1 — Arrays & Memory

Q1.A.1 — Contiguous Memory
Contiguous memory means data is stored in one single block, side by side. Arrays use this because if you know where the first item is, you can find any item using simple math (base address + index). This gives O(1) access time — super fast.
Q1.A.2 — Memory Leaks
A memory leak happens when you take memory from the system but forget to give it back. Example: you use new int[100] in a function but never call delete[]. The memory stays locked but you lost the address.
Q1.A.3 — Static vs. Dynamic Arrays
Static arrays have a fixed size set when you write the code. Dynamic arrays can change size while the program runs using new. Fixed size matters because if the array is full, you cannot add more data.
Q1.A.4 — Why Array Access is O(1)
Array access is O(1) because of direct math: address = base + (index × size). The CPU calculates the address and jumps directly. Linked Lists don't do this — nodes are scattered in memory, so you must walk one by one from the head. That's O(n).

// Linear Search — O(n) time complexity
int linearSearch(int arr[], int size, int key) {
    for (int i = 0; i < size; i++) {
        if (arr[i] == key) {
            return i;  // found at index i

Question 2 — Bubble Sort

Q2.A.1 — How Bubble Sort Works
Bubble Sort compares two side-by-side numbers. If they are in wrong order, it swaps them. In each pass, the largest unsorted number "bubbles up" to its correct position at the end. It's called Bubble Sort because smaller numbers slowly bubble their way to the top.
Q2.A.2 — Time Complexity
Best case O(n) — when array is already sorted and we use a flag to stop early. Average and worst case O(n²). Without the optimization flag, even a sorted array takes O(n²). With the flag, it becomes O(n) because we stop after one pass if no swaps happen.
Q2.A.3 — Bubble Sort vs. Selection Sort
Selection Sort is generally preferred because it does at most n swaps total. Bubble Sort can do up to n² swaps. Use Bubble Sort when data is nearly sorted or when you need a stable sort (keeps equal elements in their original order).

bubble_sort.cpp 
// Bubble Sort with early termination optimization
void bubbleSort(int arr[], int size, int &swapCount) {
    swapCount = 0;
    for (int i = 0; i < size - 1; i++) {
        bool swapped = false;
        for (int j = 0; j < size - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapCount++;
                swapped = true;
            }
        }
        if (!swapped) break;
    }
}

Question 3 — Binary Search

Q3.A.1 — Prerequisite for Binary Search
The array must be sorted. This matters because Binary Search looks at the middle element and decides whether to search the left half or right half. If the array isn't sorted, this decision makes no sense and gives wrong results.
Q3.A.2 — Dry Run with key = 23
Array: {2, 5, 8, 12, 16, 23, 38, 45}

Step 1: low=0, high=7, mid=3, arr[3]=12. 23 > 12, so we search right. low becomes 4.
Step 2: low=4, high=7, mid=5, arr[5]=23. Found at index 5!
Q3.A.3 — Why Binary Search is Faster
Binary Search is O(log n), Linear Search is O(n). Binary Search cuts the problem in half each time. For 1 million items, Binary Search needs ~20 steps. Linear Search might need 1 million. That's the difference between instant and waiting.

binary_search.cpp 
// Iterative Binary Search — O(log n)
int binarySearch(int arr[], int size, int key, int &iterations) {
    int low = 0;
    int high = size - 1;
    iterations = 0;
    while (low <= high) {
        iterations++;
        int mid = low + (high - low) / 2;
        if (arr[mid] == key) {
            return mid;  // found
        } else if (arr[mid] < key) {
            low = mid + 1;  // search right half
        } else {
            high = mid - 1;  // search left half
        }
    }
    return -1;  // not found
}

Question 4 — Complexity Analysis

Q4.A.1 — What is Big O
Big O tells us how much an algorithm's work grows when input gets bigger. We don't use seconds because that depends on the computer. Big O describes the growth pattern, which is the same on any machine.
Q4.A.2 — Big O, Omega, Theta
Think of waiting for a bus. Big O (worst case): "Bus comes in 30 min max." Big Omega (best case): "Bus could come in 2 min." Big Theta (tight bound): "Bus always comes in 10-15 min." Theta means we know both upper and lower bounds are the same.
Q4.A.3 — Ranking (Fastest to Slowest)
O(1) > O(log n) > O(n) > O(n log n) > O(n²)

Constant is fastest. Quadratic is slowest. For large inputs, even a small difference in Big O matters a lot.
Q4.A.4 — Space Complexity
Space complexity is how much extra memory an algorithm needs. If a function creates a temporary array of size n, its space complexity is O(n) because the memory used grows directly with input size.
Q4.A.5 — Faster Computer Myth
Wrong. A fast computer helps for small inputs. But when n gets really big, O(n log n) will always beat O(n²) because n² grows much faster. Computer speed cannot fix bad algorithm growth. This is called asymptotic analysis.

// Check if all elements of A exist in B
// Time Complexity: O(n²)
// Better approach: sort B(O(n log n)), then binary search each A element.
// Best approach: use a hash set for O(n) total time.
bool allElementsExist(int A[], int B[], int n) {
    for (int i = 0; i < n; i++) {
        bool found = false;
        for (int j = 0; j < n; j++) {
            if (A[i] == B[j]) {
                found = true;
                break;
            }
        }
        if (!found) return false;
    }
    return true;
}

Question 5 — Singly Linked List

Q5.A.1 — What is a Singly Linked List
A Singly Linked List is a chain of nodes where each node holds data and a pointer to the next node. It's like a treasure hunt — each clue points to the next one. Arrays store everything in one continuous block (like a row of lockers), but linked lists store nodes scattered around memory, connected by pointers.
Q5.A.2 — Time Complexities
Insert at head: O(1) — just create a node and update the head pointer.
Insert at tail: O(n) — must walk through all nodes to reach the end.
Delete by value: O(n) — must search through nodes to find the value first.
Q5.A.3 — Memory Leak in Linked Lists
If you remove a node without calling delete on it, that memory stays allocated but you can never use it again. This is called a memory leak. Always call delete on the node before removing it from the list.

void insertAtHead(int val) {
    Node* newNode = new Node(val);
    newNode->next = head;
    head = newNode;
}

// Insert at tail — O(n)
void insertAtTail(int val) {
    Node* newNode = new Node(val);
    if (head == nullptr) { head = newNode; return; }
    Node* temp = head;
    while (temp->next != nullptr) temp = temp->next;
    temp->next = newNode;
}

// Display list — O(n)
void display() {
    Node* temp = head;
    while (temp != nullptr) {
        cout << temp->data << " -> ";
        temp = temp->next;
    }
    cout << "NULL" << endl;
}

// Delete first node with given value — O(n)
void deleteByValue(int val) {
    if (head == nullptr) return;
    // Delete head
    if (head->data == val) {
        Node* toDelete = head;
        head = head->next;
        delete toDelete;
        return;
    }
    // Find and delete
    Node* temp = head;
    while (temp->next != nullptr && temp->next->data != val)
        temp = temp->next;
    if (temp->next == nullptr) {
        cout << "Value not found" << endl;
        return;
    }
    Node* toDelete = temp->next;
    temp->next = temp->next->next;
    delete toDelete;
}

▶ Q5: Linked List 
struct Node {
    int data; Node* next;
    Node(int val) { data = val; next = nullptr; }
};

class LinkedList {
public:
    Node* head;
    LinkedList() { head = nullptr; }

    void insertAtHead(int val) {
        Node* n = new Node(val);
        n->next = head; head = n;
    }

    void insertAtTail(int val) {
        Node* n = new Node(val);
        if (!head) { head = n; return; }
        Node* t = head;
        while (t->next) t = t->next;
        t->next = n;
    }

    void deleteByValue(int val) {
        if (!head) return;
        if (head->data == val) {
            Node* d = head; head = head->next; delete d; return;
        }
        Node* t = head;
        while (t->next && t->next->data != val) t = t->next;
        if (!t->next) return;
        Node* d = t->next; t->next = t->next->next; delete d;
    }

    void display() {
        Node* t = head;
        while (t) { cout << t->data << " -> "; t = t->next; }
        cout << "NULL" << endl;
    }
};
